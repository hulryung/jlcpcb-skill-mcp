jlcpcb-skill-mcp

MCP Server + Claude Skill

jlcpcb-skill-mcp_

KiCad 회로도에서 JLCPCB 조립 부품 선정까지, 대화 한 번으로 끝냅니다.

BOM 분석, 부품 추천, 조립 비용 견적을 Claude가 MCP 툴로 직접 수행합니다. 티어와 로딩피를 이해하는 추천 엔진이 재고와 가격까지 따져서 근거와 함께 제안합니다.

  • 7개 MCP 툴
  • 티어 · 로딩피 자동 계산
  • 재고 5–10× 휴리스틱
  • KiCad 6–9

Problem

왜 필요한가

JLCPCB 조립 주문에서 부품 선정은 단순한 검색 문제가 아닙니다. 수수료 구조, 재고 변동, 방대한 카탈로그가 겹치면 시제품 한 번에도 비용과 시간이 새어 나갑니다.

FEE / 로딩피

로딩피가 조용히 불어납니다

Extended 부품에는 종류당 $3의 로딩피가 붙습니다. 부품 20종이면 보드 수량과 무관하게 $60 — 같은 값이라도 Basic 부품으로 바꾸면 그대로 사라지는 비용입니다.

STOCK / 재고

재고는 발주 직전에 사라집니다

재고가 필요 수량에 겨우 닿는 부품은 며칠 사이 품절되곤 합니다. 여유 배수 없이 고르면 재발주 지연과 BOM 재작업 위험을 떠안게 됩니다.

CHOICE / 선택 피로

60만 개 중 하나를 골라야 합니다

LCSC 카탈로그에는 60만 종이 넘는 부품이 있고, 10kΩ 0603 저항만 해도 수백 종입니다. 티어·재고·가격을 매번 손으로 비교하는 일은 사람이 오래 할 작업이 아닙니다.

Architecture

동작 구조

회로도 파일이 추천 BOM이 되기까지의 흐름입니다. 모든 단계는 MCP 툴로 노출되어 Claude가 필요한 부분만 골라 호출할 수도 있습니다.

Tools

MCP 툴 레퍼런스

Claude가 호출하는 7개의 툴입니다. 모든 툴은 사람이 읽을 요약 한 줄과 기계가 읽을 JSON 블록을 함께 반환합니다.

툴주요 입력설명
search_parts query, package?, tier?, min_stock?, limit? 자유 텍스트로 부품을 검색합니다. 티어·최소 재고 필터를 지원합니다.
search_passives kind, value, package?, limit? "10k", "100nF" 같은 값 표기를 해석해 저항·커패시터를 파라메트릭 검색합니다.
get_part lcsc LCSC 번호로 단일 부품의 상세 정보(가격 구간, 속성, 재고, 티어)를 조회합니다.
find_alternatives lcsc, limit? 지정한 부품과 같은 사양·패키지의 대체 부품을 찾아 순위를 매깁니다.
analyze_kicad path .kicad_sch 회로도 또는 BOM CSV를 파싱해 값·패키지·수량 기준의 BOM 라인으로 정리합니다.
suggest_bom_parts path | bom_lines, board_qty?, stock_multiple?, max_candidates? BOM 전체에 대해 라인별 후보 추천과 비용 견적을 한 번에 수행합니다.
estimate_assembly_cost parts[{lcsc, qty_per_board}], board_qty 확정된 부품 목록의 부품비와 로딩피를 합산해 총액·장당 비용을 계산합니다.

Heuristics

선정 휴리스틱

추천 엔진과 jlcpcb-parts 스킬에 내장된 선정 기준입니다. JLCPCB의 부품 티어 구조가 출발점입니다.

티어로딩피설명
Basic 없음 약 700종의 범용 부품. 픽앤플레이스 라인에 상시 장착되어 있어 추가 비용이 전혀 없습니다.
Preferred 면제 Preferred Extended 부품. Economic PCBA 기준으로 로딩피가 면제됩니다.
Extended 종류당 $3 주문마다 부품 종류 단위로 부과됩니다. 수량이 아니라 종류 기준이라, 종류 수를 줄이는 것이 곧 절감입니다.
재고 ≥ 보드 수 × 보드당 수량 × 5–10

재고 스냅샷은 실제와 시차가 있으므로 필요 수량의 5–10배 이상 재고를 가진 부품만 신뢰합니다. 수동소자는 재고 1만 개 이상을 선호합니다.

  • 패키지 통합 — 같은 값의 저항·커패시터가 0402/0603/0805에 흩어져 있으면 한 패키지로 통합합니다. BOM 라인 수와 로딩피가 함께 줄어듭니다.
  • 가격 구간 인지 — 단가는 항상 실제 필요 수량 기준의 가격 구간으로 비교합니다. 1개 단가가 싼 부품이 100개 단가에서는 역전되기도 합니다.
  • 대체 부품 확보 — 핵심 라인에는 find_alternatives로 백업 부품을 1개 이상 확보합니다. 발주 시점 품절에 대비하는 가장 싼 보험입니다.
  • needs_review 존중 — 텍스트 검색으로만 매칭된 IC와 커넥터는 자동 확정하지 않고 검토 대상으로 표시합니다. 핀맵과 풋프린트는 사람이 최종 확인해야 합니다.

Quickstart

빠른 시작

Claude Code 기준 설정 방법입니다. 빌드 후 MCP 서버로 등록하면 스킬까지 함께 동작합니다.

  1. 빌드

    저장소를 받은 뒤 의존성을 설치하고 빌드합니다.

  2. MCP 서버 등록

    프로젝트 루트의 .mcp.json에 서버를 추가합니다. CLI를 쓴다면 claude mcp add jlcpcb-parts -- node dist/index.js 한 줄로도 등록됩니다.

  3. 스킬 사용

    함께 제공되는 jlcpcb-parts 스킬 덕분에, "이 KiCad 보드의 JLCPCB 부품을 골라 주세요"라고 요청하면 분석 → 추천 → 견적 워크플로가 자동으로 실행됩니다.

FAQ

자주 묻는 질문

비공식 API를 쓰는데 위험하지 않습니까?

jlcsearch.tscircuit.com은 커뮤니티가 운영하는 비공식 API로, 예고 없이 변경되거나 중단될 수 있습니다. 클라이언트에 타임아웃·재시도·5분 캐시를 두었고, 파라메트릭 검색이 실패하면 텍스트 검색으로 자동 폴백합니다. 최종 발주 판단은 반드시 JLCPCB 공식 정보로 하시기 바랍니다.

재고 수치는 얼마나 정확합니까?

스냅샷 기반 데이터라 실제 재고와 시차가 있습니다. 그래서 엔진이 5–10배 여유 배수를 기본으로 적용하며, 그럼에도 주문 직전에는 JLCPCB 사이트에서 재고를 다시 확인하는 것을 전제로 설계되어 있습니다.

LCSC에서 직접 단품을 구매하는 것과는 어떤 관계입니까?

이 도구의 티어와 로딩피 계산은 JLCPCB PCBA(조립) 주문 기준입니다. LCSC 단품 구매에는 로딩피 개념이 없고 가격·최소 수량 정책도 다르므로, 단품 구매 견적 용도로는 사용하지 마십시오.

어떤 KiCad 버전을 지원합니까?

KiCad 6–9의 .kicad_sch s-expression 포맷을 파싱하며, KiCad에서 내보낸 BOM CSV도 읽을 수 있습니다. 전원 심볼과 DNP·in_bom 제외 부품은 자동으로 걸러집니다.
