JLCPCB Parts MCP + Skill

KiCad 회로에 맞는 JLCPCB/LCSC 부품을 조립 비용까지 고려해 추천합니다.

Claude Code에서 "이 회로에 맞는 부품 골라줘" 한 마디면 회로도(.kicad_sch)를 분석해 LCSC 부품번호를 매칭하고, 수수료 없는 basic 부품을 우선으로 재고·가격을 검토한 뒤 추천 BOM과 예상 비용을 정리해 줍니다. 확신이 부족한 매칭은 자동 승인하지 않고 확인 요청으로 표시합니다.

설치하는 방법

Claude Code (권장)

Claude Code 안에서 두 줄이면 스킬과 MCP 서버가 함께 설치됩니다. 클론이나 빌드는 필요 없습니다.

/plugin marketplace add hulryung/jlcpcb-skill-mcp
/plugin install jlcpcb-parts@jlcpcb-tools

Codex CLI

서버 번들 파일 하나를 내려받아 등록합니다 (Node 18 이상 필요).

curl -fsSL -o ~/.jlcpcb-parts.mjs \
  https://raw.githubusercontent.com/hulryung/jlcpcb-skill-mcp/main/dist-plugin/index.mjs
codex mcp add jlcpcb-parts -- node ~/.jlcpcb-parts.mjs

등록 확인은 Codex 입력창에서 /mcp를 입력하면 됩니다.

기타 MCP 클라이언트

Cursor, Claude Desktop, Gemini CLI 등 stdio MCP를 지원하는 클라이언트라면 위에서 받은 번들 파일을 같은 방식으로 등록하면 됩니다 — command: node, args: [번들 경로].

부품 데이터 소스 (선택)

기본값은 공개 jlcsearch API라 별도 설정이 필요 없습니다. 팀에서 공유하거나 오프라인으로 쓰고 싶다면 두 가지 선택지가 있습니다.

호스팅 카탈로그 API — 다운로드 없이 (권장)

JLCPCB 재고 부품 전체(약 693,000개)를 Cloudflare D1에 올려 둔 서버입니다. 팀원은 아무것도 받지 않고 URL만 지정하면 됩니다. 등록 시 환경변수 하나만 더 넣으세요:

KiCad 파일 분석은 로컬에서 돌고 부품 데이터만 서버에서 옵니다. 최신 재고와 preferred 티어는 발주 직전 get_part가 라이브로 확인합니다.

로컬 카탈로그 DB — 완전 오프라인

전체 카탈로그를 내려받아 로컬에서 검색합니다(약 830MB 다운로드, ~4.9GB). rate limit 없이 가장 완전한 결과를 얻습니다.

사용하는 방법

KiCad 프로젝트 디렉토리에서 Claude Code를 열고 평소처럼 말하면 됩니다.

  • "이 회로에 맞는 부품 골라줘. 50장 생산 기준으로." — 전체 BOM 추천 + 비용
  • "C25804 대체품 찾아줘" — 동일 스펙 대체 부품 탐색
  • "이 부품 목록으로 100장 조립 비용 계산해줘" — 부품비 + 로딩피 산출
  • "추천 부품을 회로도에 지정해줘" — 회로도에 LCSC 필드 직접 기록

결과는 라인별 추천 표(LCSC 번호·티어·재고·단가)와 비용 요약, 리스크 목록으로 정리됩니다.

알아두면 좋은 것

· 회로도를 수정한 뒤에는 KiCad에서 F8(Update PCB from Schematic)로 PCB에 반영하세요.

· 재고와 가격은 수시로 바뀝니다 — 실제 주문 직전에 "재고 다시 확인해줘"로 재검증하세요.

· "확인 필요(needs_review)"로 표시된 부품은 데이터시트에서 풋프린트를 직접 확인한 뒤 쓰세요.
